# Todoアプリ 要件定義

## 概要
個人用のWebベースTodoリストアプリケーション。

- **プラットフォーム**: Webアプリ
- **バックエンド**: Python / Flask
- **データベース**: SQLite
- **フロントエンド**: HTML / CSS（シンプル）

---

## 機能要件

### 1. タスク管理

#### タスクの情報
| フィールド | 説明 |
|---|---|
| タスク名 | 必須 |
| 期限 | 日付（必須） |
| カテゴリ | タブに対応（必須） |
| 優先度 | 期限から自動計算（高・中・低） |
| 完了フラグ | 完了/未完了 |
| 作成日時 | 自動付与 |

#### CRUD操作
- タスクの追加
- タスクの編集
- タスクの削除
- タスクの完了チェック

### 2. 優先度の自動計算

期限日から現在日を引いた残り日数で自動判定する（固定値）。

| 残り日数 | 優先度 |
|---|---|
| 0日以下（期限切れ含む） | 高 |
| 1〜2日 | 高 |
| 3〜7日 | 中 |
| 8日以上 | 低 |

- 優先度はユーザーが手動設定するのではなく、常に自動計算で表示する
- 期限切れのタスクは「高」として表示し続ける

### 3. カテゴリ（タブ）

- カテゴリはユーザーが追加・削除できる（デフォルトで「仕事」「買い物」「副業」を用意）
- 「すべて」タブを常に表示し、全カテゴリのタスクを一覧表示する
- カテゴリを削除する際、そのカテゴリに属するタスクも削除する（要確認）

### 4. 表示・並び替え

- **デフォルト**: 期限順（昇順）
- 以下の並び替えに切り替え可能：
  - 期限順
  - 優先度順
  - 作成日順

- 完了済みタスクは通常一覧に表示しない
- 「完了済み」タブで完了したタスクを後から確認できる

---

## 非機能要件

- シングルユーザー前提（ログイン機能なし）
- レスポンシブ対応は必須としない
- データはSQLiteファイルでローカル保存

---

## 確認済み事項

| # | 項目 | 決定内容 |
|---|---|---|
| 1 | カテゴリ削除時、タスクはどうするか | タスクも一緒に削除 |
| 2 | 優先度の日数しきい値を変更可能にするか | 固定値 |
| 3 | 完了済みタスクを後から確認したいか | 「完了済み」タブで確認可能 |
